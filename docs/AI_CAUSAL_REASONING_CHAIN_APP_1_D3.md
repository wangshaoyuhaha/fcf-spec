# AI-CAUSAL-REASONING-CHAIN-APP-1 D3

## Stage

AI-CAUSAL-REASONING-CHAIN-D3

## Purpose

Construct deterministic causal chains from registered D2 causal
claim records.

D3 does not determine whether any causal claim is true.

## Chain construction

Each chain preserves:

- registered causal claim records
- cause and effect references
- registered premises
- supporting evidence references
- counterevidence references
- alternative explanation references
- correlation identifier
- research run identifier
- source record status
- operator review requirement

## Deterministic structure

D3 creates:

- canonical claim ordering
- canonical edge ordering
- canonical node identifiers
- root node identifiers
- terminal node identifiers
- deterministic chain status
- deterministic reason codes

D3 does not use model-generated ordering or probability.

## Chain states

A chain may be:

- READY_FOR_ASSESSMENT
- REVIEW_REQUIRED
- BLOCKED

A READY_FOR_ASSESSMENT chain is not a confirmed causal chain.

It only means the registered claim records are structurally ready for
D4 governance assessment.

## Interpretation boundary

Every chain retains:

- causal truth status: UNDETERMINED
- probability status: NOT_ASSIGNED
- winner status: NOT_SELECTED
- operator review status: REQUIRED
- source artifacts: PRESERVED
- original conclusions: PRESERVED

## Deferred to D4

D4 will assess:

- disconnected chain components
- cycles
- duplicated or conflicting edges
- missing registered evidence
- missing counterevidence review
- missing alternative explanations
- contradiction signals

D3 does not resolve any of these automatically.
