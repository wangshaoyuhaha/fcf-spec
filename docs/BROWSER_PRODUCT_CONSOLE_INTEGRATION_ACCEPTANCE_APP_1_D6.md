## Windows capacity rejection repair

The bounded concurrency rejection path now drains a bounded request header,
sends the complete HTTP 503 response, half-closes the write side, and then uses
the existing server cleanup path. This prevents Windows from resetting the
connection before the deterministic rejection response is delivered.

The repair does not increase capacity, weaken rejection, add mutation, or
change the loopback-only and read-only security boundary.
