# Controlled Learning and Deterministic Backtesting P0-P3 Stage 11

## P0 - Point-in-Time and Version Foundation

- data-source version locks and content digests
- point-in-time evidence with event, publication, availability, ingestion,
  as-of, and retrieval timestamps
- market calendar, corporate action, benchmark, universe, strategy, factor,
  portfolio, policy, schema, model-role, Prompt, cost, tax, slippage, funding,
  liquidity, roll, and experiment configuration snapshots
- mandatory 'available_at <= as_of_time'

## P1 - Deterministic Backtest and Outcome Foundation

- train, validation, and final-test separation
- purged validation and embargo controls
- chronological walk-forward overlap checks
- deterministic returns, errors, costs, slippage, liquidity, capacity, regime,
  factor, failure, and portfolio metrics
- immutable positive, negative, blocked, failed, and inconclusive results
- Outcome labels preserve original prediction, observation time, data version,
  split, and evaluation policy
- append-only result registry

## P2 - Registered AI Historical Evaluation

- HISTORICAL_REPRODUCTION and COUNTERFACTUAL_CURRENT_MODEL_EVALUATION identities
- registered model, Prompt, output, evidence, and training-cutoff metadata
- knowledge-leakage, fact-alignment, model-role, and incremental-value checks
- no model invocation or Prompt execution
- AI remains advisory

## P3 - Controlled Learning and Evolution

- explicit human feedback
- nine governed candidate types
- declared single-variable or compound changes
- fourteen static qualification checks
- Champion/Challenger sandbox comparison
- eighteen evolution gates
- new-version promotion proposal and preserved Champion rollback target
- deterministic learning-loop audit digest
- eligible results stop at Operator review
- no automatic activation, promotion, Champion replacement, or rollback

## D6 Acceptance

All 22 canonical P0-P3 capabilities retain independent identities in the
capability registry and are displayed through registered Web Console sections.
