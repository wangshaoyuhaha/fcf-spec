# FCF Web Console Stage 8 D1-D6

## Scope

'FCF-WEB-CONSOLE-APP-1' delivers the custom browser product surface required
by the FCF V2 architecture. It composes existing registered Browser Console
evidence into a governed product interface without changing frozen P1-P47.

## D1 - Boundary and Product Contract

- immutable paper-only, local-only, loopback-only, sidecar-only boundary
- Registered Evidence and Deterministic Engine authority preserved
- AI advisory only
- twelve product routes covering intake, research, monitoring, evidence,
  models, risk, portfolio, Paper portfolio, reports, review, and operations

## D2 - Governed Intake Requests

- PDF, Excel, CSV, JSON, text, approved HTTPS URL, local-file, and multi-file
  descriptors
- checksum, size, extension, loopback, explicit confirmation, credential-like
  metadata, source, trust, freshness, and licensing validation
- validated inputs remain quarantined pending approved evidence registration
- URL input records a reference and performs no retrieval

## D3 - Controlled Requests

- controlled research question and explanation requests
- approval, rejection, re-analysis, more-evidence, untrusted-data, comparison,
  override, freeze, export, start, and stop requests
- explicit Operator identity, target, reason, confirmation, and request digest
- validation never causes an automatic transition or financial execution

## D4 - Operational Visibility

- workflow state, registered model and Prompt metadata, retries, failures,
  degradation, cost, disagreement, risk, policy, and review presentation
- immutable Mapping payload rendering
- health endpoint identifies every authority boundary

## D5 - Product Experience

- responsive custom HTML/CSS browser shell
- Portfolio Construction and deterministic stress presentation
- Paper position proposal and comprehensive report presentation
- Operator review controls with explicit reason and confirmation
- product-visible Stage 9 boundary for local process lifecycle packaging

## D6 - Integration Acceptance

- existing 'ConsoleReadModel' integration
- GET, HEAD, and governed POST dispatch contracts
- hardened exact-loopback HTTP adapter with bounded JSON request bodies
- browser-side checksums and working intake, research, review, start, and stop
  request submission
- exact '127.0.0.1' peer enforcement
- fail-closed method, route, payload, confirmation, and authority behavior

## Deferred

- actual process launch, stop, supervision, and nontechnical packaging belong
  to 'ONE-CLICK-LOCAL-OPERATIONS-APP-1'
- multi-market Paper and Shadow validation
- controlled learning and deterministic backtesting P0-P4
- live retrieval, model invocation, Prompt execution, credentials, accounts,
  wallets, brokers, exchanges, positions, balances, orders, or execution
