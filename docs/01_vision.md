# D1 - FCF 项目愿景

## 1. 项目定位

FCF 不是单一策略，也不是简单的聊天机器人。

FCF 是一个以事件、契约、风控、回放和影子模式为核心的决策系统框架。

它的目标是让复杂决策过程可以被记录、审计、复盘、替换和持续改进。

## 2. 核心目标

FCF 的核心目标是：

- 先建立可持续演进的系统骨架
- 再逐步接入数据、模型、策略和执行模块
- 所有决策都必须可追踪
- 所有结果都必须可回放
- 系统不确定时，必须停止影响真实资金
- 实盘可以停止，但学习不能停止

## 3. 设计方向

FCF 遵守以下方向：

- Contract First：先定义契约，再写业务逻辑
- Event Driven：所有模块通过事件通信
- Replayable：所有决策过程都能回放
- Risk First：风控优先于收益
- Shadow First：不确定时进入影子模式
- Replaceable：每个模块都可以替换

## 4. 当前项目阶段

当前项目处于 Phase 1：Build Spine。

Phase 1 的重点不是追求收益，而是搭建系统主干。

当前主干包括：

- 项目仓库
- 项目状态文件
- 架构文档
- 事件契约文档
- 后续可回放、可审计、可扩展的基础

## 5. 阶段路线

项目初步分为四个阶段：

1. Phase 1：Build Spine，搭建系统骨架
2. Phase 2：Replace Organs，替换和接入核心模块
3. Phase 3：Replay Evolution，通过回放持续改进
4. Phase 4：Adaptive Learning，形成自适应学习系统
